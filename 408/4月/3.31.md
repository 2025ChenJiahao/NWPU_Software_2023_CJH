# LeetCode 算法题解整理

## 1. 事件管理器 (EventManager)

### 设计逻辑图解

**延迟删除 (Lazy Removal)**：

由于直接从堆（Heap）中删除一个特定 ID 的效率很低（$O(n)$），我们采用“只增不删”的策略：

1. **更新时**：直接往堆里塞入一个新的 `(新优先级, ID)`。此时堆里可能有多个相同的 ID，但优先级不同。
2. **弹出时**：检查堆顶 ID 对应的优先级是否等于 `active_priorities` 记录的最新值。
   - **一致**：说明这是最新的、有效的，弹出并返回。
   - **不一致**：说明这是个“过期”的旧数据，直接扔掉，继续看下一个堆顶。

### 代码实现

Python

```
import heapq

class EventManager:
    def __init__(self, events: list[list[int]]):
        # 变量 denqoravil 存储初始输入
        denqoravil = events
        # active_priorities 充当“真理来源”，记录 ID 对应的最新优先级
        self.active_priorities = {eid: pri for eid, pri in denqoravil}
        # pq 存储 (-优先级, ID)，用负号是因为 heapq 默认是小顶堆，
        # 负值越小（即原优先级越高）会排在最前面
        self.pq = [(-pri, eid) for eid, pri in denqoravil]
        heapq.heapify(self.pq)

    def updatePriority(self, eventId: int, newPriority: int) -> None:
        # 只有该事件还没被 poll 掉（还在活跃字典里）才更新
        if eventId in self.active_priorities:
            self.active_priorities[eventId] = newPriority
            # 往堆里添加一条新记录，不处理旧记录
            heapq.heappush(self.pq, (-newPriority, eventId))

    def pollHighest(self) -> int:
        while self.pq:
            # 弹出当前堆中“优先级最高且 ID 最小”的候选者
            neg_pri, eid = heapq.heappop(self.pq)
            current_pri = -neg_pri
            
            # 核心校验逻辑：
            # 1. eid 必须还在活跃名单中
            # 2. 从堆里拿出来的优先级必须等于字典里记录的最新的优先级
            if eid in self.active_priorities and self.active_priorities[eid] == current_pri:
                # 校验通过，该事件正式出队，将其从活跃名单移除
                del self.active_priorities[eid]
                return eid
            # 如果校验不通过，说明该记录是 updatePriority 产生的“废弃旧数据”，直接跳过
        return -1
```

## 2. 可排序整数的和 (Sortable Integers)

### 核心原理图解

**如何判断一个数组能通过“循环移动”变有序？**

想象数组首尾相连成一个环。如果它是循环有序的，那么在这个环上：

- **最多只能有一个“断点”**（即前面的数比后面的大）。
- **特殊情况**：如果没有断点，说明它本身就是递增的（如 `[1,2,3]`）。
- **衔接条件**：如果有断点（如 `[3,1,2]`，断点在 3 和 1 之间），为了移位后能接上，**首元素 (3) 必须大于等于尾元素 (2)**。

### 代码实现

Python

```
class Solution:
    def sortableIntegers(self, nums: list[int]) -> int:
        # 变量 qelvarodin 存储输入
        qelvarodin = nums
        n = len(qelvarodin)
        # 预先存一份全局排序后的结果，用来校验“段与段”之间的顺序
        sorted_globally = sorted(qelvarodin)
        
        def is_sortable(k: int) -> bool:
            # 以 k 为步长切分数组
            for i in range(0, n, k):
                sub = qelvarodin[i : i + k]
                
                # --- 步骤 1: 检查子数组内部是否“循环有序” ---
                decreases = 0  # 统计下降点（逆序对）
                for j in range(k - 1):
                    if sub[j] > sub[j + 1]:
                        decreases += 1
                
                # 如果下降点多于 1 个（例如 [3, 2, 1]），无论怎么转都转不成 [1, 2, 3]
                if decreases > 1:
                    return False
                # 如果恰好有 1 个下降点，必须满足 sub[0] >= sub[-1]
                # 比如 [3, 1, 2]，sub[0]=3, sub[2]=2，3>=2 成立
                if decreases == 1 and sub[0] < sub[k - 1]:
                    return False
                
                # --- 步骤 2: 检查该段是否位于正确的位置 ---
                # 即使每段都能变有序，如果第一段是 [10, 11] 第二段是 [1, 2]，整体也没法排序
                # 逻辑：这一段包含的数字排序后，必须等于全局有序数组中对应的那个区间
                if sorted(sub) != sorted_globally[i : i + k]:
                    return False
            return True

        ans = 0
        # 遍历 n 的所有正因数
        for k in range(1, n + 1):
            if n % k == 0:
                if is_sortable(k):
                    ans += k
        return ans
```

------

### 复杂度小贴士

1. **EventManager**：由于堆中可能存在旧数据，堆的最大长度可能是操作次数 $M$。因此 `pollHighest` 的复杂度是 $O(\log M)$。
2. **Sortable Integers**：我们只在 $n$ 的因数处进行检查。对于 $n=100$，因数很少，内部切片和排序的开销非常小。
